## 1. 개요

### 1.1 목적

경력 4년 5개월의 프론트엔드 개발자로서 기술적 깊이와 설계 역량을 시각적으로 전달하는 포트폴리오 사이트 제작.
경력기술서의 텍스트 내용을 보완하는 **시각적 증명 도구**로 활용.

### 1.2 목표 독자

- **채용 담당자** — 빠르게 스캔, 인상적인 첫 화면이 중요
- **기술 면접관** — 프로젝트 상세 및 기술적 의사결정 확인

### 1.3 성공 기준

- 채용 담당자가 1분 안에 핵심 경력과 기술 스택 파악 가능
- 각 프로젝트의 "왜 이 결정을 했는가"가 전달되는 수준
- 모바일/데스크탑 모두 자연스럽게 동작

## 3. 사이트 구조

싱글 페이지 애플리케이션 (SPA). React Router 미사용, 해시 기반 앵커 내비게이션.

```
┌─ RetroBrowser (레트로 브라우저 프레임)
│  ├─ Titlebar       # 빨/노/초 dots + 제목 + 테마 토글
│  ├─ Navbar          # 스티키 내비게이션 (해시 링크)
│  ├─ Hero            # 아바타, 인사, 이름, 타이핑 애니메이션, CTA
│  ├─ About           # 경력 요약, 정보 그리드, 자격증, 수상
│  ├─ Skills          # 카테고리별 기술 스택 그리드
│  ├─ Projects        # 프로젝트 카드 그리드 + 상세 모달
│  ├─ Experience      # 경력 타임라인
│  ├─ Contact         # 연락처 + GitHub 링크
│  └─ Footer          # 저작권 + 맨 위로 버튼
└─ ProjectModal       # 오버레이 모달 (AnimatePresence)
```

## 4. 디자인 방향

### 4.1 컨셉: 레트로 90년대 브라우저

기존 `my-profile-site`의 **레트로 브라우저 스타일** 컨셉을 그대로 유지한다.

**핵심 디자인 요소:**
- **브라우저 프레임** — 전체 페이지를 감싸는 레트로 윈도우 프레임. 타이틀바에 빨/노/초 dots.
- **각진 테두리** — `border-radius: 0` 전면 적용. 둥근 모서리 없음.
- **픽셀 오프셋 그림자** — `box-shadow: Npx Npx 0 0 var(--pixel-shadow)` 패턴. 호버 시 오프셋 증가, 클릭 시 감소.
- **픽셀 폰트** — 제목/라벨에 `Press Start 2P`, 본문에 `IBM Plex Mono` / `Pretendard`.
- **체크무늬 배경** — `radial-gradient` 도트 패턴 (12px 간격).
- **대시 디바이더** — 섹션 구분선에 `repeating-linear-gradient` 대시 패턴.
- **스캔라인 오버레이** — Hero 영역의 CRT 모니터 느낌 수평선.

### 4.2 테마 시스템

CSS 변수 기반 라이트/다크 모드 전환.

| 속성 | 라이트 | 다크 |
|------|--------|------|
| 외부 배경 | `#f5a68c` (살구색) | `#1a0a2e` (딥 퍼플) |
| 윈도우 배경 | `#e8e8e8` | `#1e1e2e` |
| 텍스트 | `#1a1a1a` | `#e8e8e8` |
| 액센트 | `#e05030` (레드) | `#ff6b4a` (오렌지) |
| 그림자 | `#222222` | `#8888aa` |

- `data-theme` 속성으로 전환 (Tailwind `dark:` 접두사 불필요)
- `prefers-color-scheme` 시스템 설정 자동 반영
- `localStorage`의 `theme-preference` 키로 수동 선택 유지

### 4.3 반응형 브레이크포인트

| 브레이크포인트 | 화면 | 주요 변화 |
|--------------|------|----------|
| > 1024px | 데스크탑 | 전체 브라우저 프레임 + 그림자 |
| 768–1024px | 태블릿 | 프레임 축소, 프로젝트 2열 유지 |
| 480–768px | 모바일 | 프레임/그림자 제거, 1열 레이아웃, 햄버거 메뉴 |
| < 480px | 소형 모바일 | 타이틀바 축소, 버튼 풀 너비 |

## 6. 인터랙션 & 애니메이션

### 6.1 Framer Motion 적용 범위

- 섹션 진입 애니메이션 (stagger fade-in-up)
- 프로젝트 모달 열기/닫기 (scale + fade)
- 모바일 메뉴 전체화면 전환
- 페이지 최초 로드 시 Hero 요소 순차 등장

### 6.2 CSS @keyframes 유지 (무한 루프)

- `pixelBounce` — 아바타 바운스
- `blink` — 타이핑 커서 깜빡임
- `scrollBounce` — 스크롤 유도 화살표

### 6.3 기타 인터랙션

- 스크롤 시 `IntersectionObserver`로 요소 등장 (`reveal` → `visible`)
- 내비게이션 활성 링크 하이라이트 (스크롤 위치 감지)
- 프로젝트 모달: ESC 키 / 오버레이 클릭 / X 버튼 닫기 + body 스크롤 잠금
- 버튼 호버: 오프셋 그림자 증가, 클릭: 오프셋 감소 (눌림 효과)

## 7. 구현 단계

### Phase 1: 프로젝트 스캐폴딩 (1~2일)

- Vite + React + TypeScript 프로젝트 초기화
- Tailwind CSS + PostCSS + Autoprefixer 설정
- CSS 변수 정의 (라이트/다크 테마 25개+)
- 폰트 CDN 로드 + 기본 레트로 스타일 적용
- 다크모드 토글 구현
- Vercel 배포 파이프라인 구성

### Phase 2: 핵심 섹션 구현 (3~5일)

- 레트로 브라우저 프레임 (RetroBrowser) 셸
- Hero 섹션 + 타이핑 애니메이션
- About 섹션 + 자격증/수상
- Experience 타임라인
- Skills 그리드

### Phase 3: 프로젝트 섹션 (3~4일)

- 프로젝트 카드 컴포넌트
- 프로젝트 상세 모달 (Framer Motion AnimatePresence)
- 실제 프로젝트 데이터 교체
- 스크린샷/GIF/아키텍처 다이어그램 삽입

### Phase 4: 마무리 (1~2일)

- 반응형 점검 (4개 브레이크포인트)
- 애니메이션 다듬기
- 성능 최적화 (이미지 압축, lazy loading)
- SEO 메타태그 설정
- 최종 배포 및 이력서 링크 반영
